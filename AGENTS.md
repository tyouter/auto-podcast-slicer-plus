# AGENTS.md

> 本文件是 AI 编码执行器（Reasonix / Claude Code 等）在本仓库做任何代码改动前**自动加载的工程铁律**。动手前必须遵守。

## 卡帕西四原则（编码执行铁律）

1. **Think Before Coding** — 不假设、不隐藏困惑；动手前先亮出 tradeoff 和不确定点。
2. **Simplicity First** — 最简代码，无投机、无过度抽象、不为想象中的未来需求提前设计。
3. **Surgical Changes** — 只碰必须碰的；不顺手重构、不优化相邻代码、不动无关文件。
4. **Goal-Driven Execution** — 定义可验证目标，每步跟验证，循环直到通过。

## 本仓库的架构边界（铁律）

- **代码库只含通用工具 + 模板**：配置模板、勘误模板、样式模板、项目目录模板。
- **具体项目的实际数据**（某个播客的勘误表、转录、切片、视频、项目专属配置）**生成 / 存放在用户指定的项目文件夹，绝不进代码库**。
- **示例数据卫生**：文档和代码示例里的数据必须是占位符（`<...>` / `/path/to/`）或明确标注的虚构数据。任何真实项目的运行数字、真实文件名、真实勘误条目、真实本地路径**不得进入仓库**。

## 改动纪律

- 改完跑 `pytest tests/` 验证；现有三入口（`run_from_audio` / `run_from_transcript` / `run_montage`）行为**不得回归**。
- **不做任何 git 操作**（不 add / 不 commit / 不 push）——改动留工作区，由人审计后手动提交。
- 每个任务**单一职责**，独立可验收，不揉合多个任务。
- 改 API 时区分破坏性变更：带默认值字段非破坏；改必填 / 删字段是 breaking，必须同步迁移所有调用点 + 更新测试，不留半成品仓库。

## 执行环境

- Windows + conda env `garden`（funasr / torch+cu118 / ffmpeg）。脚本内路径用 `D:/` 或 `D:\` 格式，不认 `/d/`。
- garden_core 是 editable install，改源码后无需重新 `pip install`。
- 环境已配好——不要 `pip install`、不要改 Python 版本 / PATH；遇导入错误是调用方式问题，不是环境问题。
